Overview

This addon allows users to do two types of control methods. The first is a single mode PID control with gain scheduling and bumbless manual control. The second is simple on and off control for industrial actuators.

Repo Location:

GitHub - NIVeriStandAdd-Ons/Control-Custom-Device​

Requirements

These drivers must be installed on the deployment computer/target to function.

  • NI VeriStand 2019 or later

Addon Specifics

Addon Configuration

Upon adding the addon, a configuration GUI will open. A previously exported addon configuration can be imported.

​​After adding the Control Add-on to the system definition, the main page will be displayed. The user can add a PID or Industrial controller by selecting the respective button at the top.

​

PID Controller

To configure the PID controller, first select Feedback and Output channels using the channel browse button.

​​

Fill in a Max and Min setpoint. These values are used to limit the setpoint value passed to the PID controller algorithm. Additionally, fill out the Output Max and Min. These are hard limits on what is passed out of the PID controller algorithm. These limits apply in the manual control mode as well.

​

Next, the user can enable and select an external setpoint. Don't use this option unless it is really necessary.

​

Last, go to the PID Paramete​​rs section and put in the default PID gains. Values will not be committed unless the user selects Apply. Additionally, the user can import PID settings from a CSV file. The CSV file format is P, I, D, GainScheduleValue. Also notice the Enable Live Updates field. Selecting this will allow the user to change (tune) the PID settings at runtime. Otherwise, the PID parameters can be viewed but not modified.

 ​

​PID gain scheduling is supported. To use this feature, select Enable Gain Scheduling and browse for the desired channel.

​

Then, on the PID Settings table, right click and select Add Row.

​​​​

This will allow the user to add more PID settings to the gain schedule.

Take the above settings as an example, if the Gain Schedule Signal was less than 100, the first row of PID gains would be used. If the signal goes above 100, the second set of gains would be used.

Industrial Controller

The industrial controller is an on off controller. If the feedback channel is above the setpoint, the controller will deactivate the output. If the feedback channel is below the setpoint, it will activate the output.​

The Industrial Controller configures like the PID controller. After adding it from the main section, select the Feedback and Output channels by using the browse button.

​Fill in the Setpoint M​ax and Min. This limits the setpoint value going to the controller. Next, set the Activated and Deactivated Levels. When the controller output is activated (feedback below setpoint), the activated level will be passed to the output channel. For example, if the actuator is a Heater relay, the value would be 1. If the actuator is PWM based output, the value be 100%. Additionally, if the user had an inverted system, the activated level would be 0 and the deactivated level be 1.

 

Last, on the industrial parameters section, the user can add some smarts to the control logic. Specifically, add a Hysteresis and Switching Filter. The Hysteresis acts like a thermostat. If the setpoint is 70 and the hysteresis is 2, the temperature must reach 72 before the controller activates the output. However, the temperature must fall to 68 before deactivating the output. The Switching Filter makes a feedback value be above the switching level for a set amount of time before the switch happens. So, in the above example, the temperature would have to be above 72 for some set time before activating and below 68 for the same amount of time before deactivating.​​​

​​​Addon Execution

Deploy the system definition. Use the linked setpoint channel and output channel for monitoring purposes. If more insight is needed, like for tuning, just the workspace object.​

Workspace Object

The addon comes supplied with two workspace objects. ​These objects are intended to be visual displays of the controller and well as a managed way to change setpoints and output values. To add the object, change the workspace to Edit mode, go to the Control section and select the desired type.

The workspace object will pop up a dialog asking the user to select the desired controller. Only one controller per workspace object can be selected.

Upon selection, the graph will start plotting Feedback, Output, and Setpoint channels. To make a change to the setpoint or manual control settings, enter the desired value and hit Apply. This helps ensure the user doesn't enter a accidental value that could damage the controlled object. If the user enables Manual Control, the Manual Output value entered will be directly passed out to the output channel. Again, the output lim​its from the system definition configuration are applied to keep the controlled object safe.​

The industrial controller workspace object works the same way but only allows the user to select the Industrial Controllers in the system.

PID Tuning Tool

In addition to the workspace object, there is a workspace tool for changing (tuning) PID gains. Add this tool to the project files:

 

​It can also be added as a button to the UI screen.

Click open on the file or select the configured button to launch the tool.​

 

Select File >> Configure and select the desired addon.​​

When the tool links to the selected Control Addon, it will load all available PID controllers and gains. Select the desired PID controller to modify (tune) from the PID Controller Name dropbox.

 

Notice the Live Updates Allowed? field. If that is not selected, the user can view the PID gains but not change them. First allow changes from the system definition (see above). If changes are allowed, make the changes as needed through the PID Settings table. The user must hit apply for the change to happen. This is to maintain safety of the controlled object.​

CRITICAL - Keeping PID Gain Changes

The user makes PID gain changes through the PID Tool (above). This happens while this system is deployed. To keep the changes permanently (on the next deployment), the user must export them from the tool as a CSV file.

​

Then the user must go back into the system definition after an undeploy and import that same CSV file as the new default settings.
